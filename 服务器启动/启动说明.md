## 二、启动步骤

### 1. 启动 API 服务器（端口 3001）

```powershell

C:\Users\Administrator\.workbuddy\binaries\node\versions\22.22.2\node.exe index.js
```

```powershell
curl http://localhost:3001/api/health

```

访问：http://localhost:5173/

## 三、一键启动（PowerShell）

运行此目录下的脚本：

```powershell
.\start-servers.ps1
```

脚本会自动：
1. 清理端口 3001/5173/5174 上的旧进程
2. 依次启动 API → 前端5173 → 管理端5174
3. 自动验证各端口是否正常

---

## 四、停止服务器

```powershell
# 查找并停止所有端口进程
$ports = @(3001, 5173, 5174)
foreach ($port in $ports) {
    $conn = Get-NetTCPConnection -LocalPort $port -ErrorAction SilentlyContinue | Select-Object -First 1
    if ($conn -and $conn.OwningProcess -gt 0) {
        Stop-Process -Id $conn.OwningProcess -Force -ErrorAction SilentlyContinue
    }
}
```

## 七、常见问题排查

### 无法登录？
- 确认 API 服务器（3001）已启动 → `curl http://localhost:3001/api/health`
- 确认 MySQL 正在运行 → `netstat -an | grep 3306`

### 页面显示异常/空白？
- 打开浏览器开发者工具（F12）→ Console 查看报错
- 确认 5173 端口返回 200 → `curl -s -o /dev/null -w "%{http_code}" http://localhost:5173/`
- 尝试刷新页面并清除缓存（Ctrl+F5）

### 端口被占用？
```powershell
# 查看占用端口的进程
Get-NetTCPConnection -LocalPort 5173

# 终止进程（替换 PID）
Stop-Process -Id <PID> -Force
```
