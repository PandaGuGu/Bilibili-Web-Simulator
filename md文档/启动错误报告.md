## 二、今日踩坑全记录

### 坑1: `npx node --watch` 启动失败

**错误命令**: `npx node --watch index.js`
**现象**: 后台任务卡住，无输出
**根因**: `npx node` 会去 npm 下载名为 `node` 的第三方包，而不是使用本机 Node.js
**正确**: `node index.js`（去掉 npx）

---

### 坑2: 后端进程反复崩溃

**现象**: 3001 端口频繁掉线（connection refused），前端全卡 loading
**根因**: 频繁 kill/restart 后端口未及时释放；MySQL 连接池在网络波动时可能挂起
**预防**: 重启后立即 curl health 验证；使用 netstat 确认端口已释放

---

### 坑3: Vite dev server HMR 缓存导致页面空白

**现象**: 编辑 .tsx 后页面白屏，但 HTTP 返回 200
**根因**: 频繁修改源码时 Vite HMR 未正确重编译
**解决**: 杀掉 5173/5174 进程，完整重启 vite dev server
**建议**: 开发期间只改代码不手动 `vite build`

---

### 坑4: TypeScript 变量声明顺序导致编译失败

**现象**: UserProfile.tsx 白屏
**错误**: `TS2448: Block-scoped variable 'favCount' used before its declaration.`
**根因**: `favCount` useState 在 L90，但 tabs 数组在 L76 引用了它
**解决**: 把所有 useState 集中在组件顶部，在派生变量之前

---

### 坑5: Zustand 持久化导致头像不更新

**现象**: 上传头像成功但页面不刷新
**根因**: 头像取自 Zustand persist → localStorage，reload 也恢复旧数据
**解决**: 上传后直接 `useStore.setState()` 更新内存数据，不依赖 reload

---

### 坑6: vite build 与 vite dev 互相干扰

**现象**: build 后 dev server 渲染异常
**建议**: 开发期只用 dev server，提交前再 build 验证

## 三、下次启动

```powershell
# 终端1 — 后端（先启）
cd server && node index.js

# 终端2 — 用户端
node node_modules/vite/bin/vite.js --port 5173

# 终端3 — 管理端
node node_modules/vite/bin/vite.js --port 5174
```

**验证三端**:
```bash
curl localhost:3001/api/health
curl -s -o /dev/null -w "%{http_code}" http://localhost:5173
curl -s -o /dev/null -w "%{http_code}" http://localhost:5174
```

**停止所有**:
```powershell
netstat -ano | findstr ":3001 :5173 :5174" | ForEach { $l=$_ -split '\s+'; taskkill /PID $l[-1] /F }
```
